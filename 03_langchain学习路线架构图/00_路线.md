## 🎯 LangChain TypeScript 学习路线总结

### 一、LangChain TS 核心架构概览

LangChain TypeScript 版本（LangChain.js）与 Python 版本功能对等，支持 Node.js 18+、Cloudflare Workers、Vercel/Next.js、Deno、Bun 等多种运行环境 。核心采用模块化设计：

```
核心包体系：
├── @langchain/core          # 核心抽象（Prompts、Messages、Runnables）
├── langchain               # 高层封装（Chains、Agents）
├── @langchain/openai       # OpenAI 集成
├── @langchain/langgraph    # 状态化 Agent 工作流
└── @langchain/community    # 社区集成（向量库、文档加载器等）
```

---

### 二、分阶段学习路线

#### **阶段 1：基础入门（1-2 周）**

**目标**：掌握基础概念，能构建简单的 LLM 调用链

| 主题 | 关键内容 | 实践项目 |
|------|----------|----------|
| **环境配置** | Node.js 18+、TypeScript 配置（ESM/CommonJS）、API Keys 管理  | 初始化项目模板 |
| **Model I/O** | ChatOpenAI、消息类型（HumanMessage/AIMessage/SystemMessage）、参数调优（temperature/maxTokens） | 多轮对话机器人 |
| **Prompts** | ChatPromptTemplate、MessagesPlaceholder、Few-shot prompting  | 动态提示生成器 |
| **Output Parsers** | StringOutputParser、StructuredOutputParser（Zod 集成）、JsonOutputParser  | 结构化数据提取 |
| **LCEL 基础** | 管道操作符 `\|`、RunnableSequence、基本链式组合  | 简单翻译/总结链 |

**关键概念**：LCEL（LangChain Expression Language）是现代推荐的链式构建方式，使用管道符 `|` 组合组件，支持流式、异步、并行执行优化 。

---

#### **阶段 2：核心组件进阶（2-3 周）**

**目标**：掌握 Memory、RAG、Agents 三大核心能力

**2.1 Memory 系统**
- **ConversationBufferMemory**：全量存储对话历史（适合短对话）
- **ConversationBufferWindowMemory**：滑动窗口保留最近 k 轮（控制 Token）
- **ConversationSummaryMemory**：自动摘要历史（长对话优化）
- **Vector Store Memory**：基于语义检索的历史查询

```typescript
// 典型集成方式
const memory = new ConversationBufferWindowMemory({
  k: 4,
  memoryKey: "history",
  returnMessages: true
});

const prompt = ChatPromptTemplate.fromMessages([
  ["system", "你是专业顾问"],
  new MessagesPlaceholder("history"),
  ["human", "{input}"]
]);
```

**2.2 RAG（检索增强生成）**
- **文档处理流程**：DirectoryLoader → TextSplitter → Embeddings → Vector Store
- **向量数据库**：Pinecone（托管）、Chroma（本地）、Supabase pgvector（Postgres 扩展）、Redis
- **检索策略**：Similarity Search、MMR（最大边际相关性）、Self-Query、Ensemble Retriever
- **完整 RAG 链**：`retriever | prompt | model | output_parser`

**2.3 Agents 与 Tools**
- **Tool 定义**：StructuredTool、DynamicTool、Zod 模式验证
- **Agent 类型**：ReAct、Plan-and-Execute、OpenAI Functions
- **执行流程**：Thought → Action → Observation → Final Answer

---

#### **阶段 3：LangGraph 状态化工作流（2-3 周）**

**目标**：构建复杂多步骤、有状态、可持久化的 AI Agent

LangGraph 是 LangChain 团队推出的低级编排框架，适合构建复杂 Agent 工作流 ：

**核心概念**：
- **State**：共享状态对象（TypedDict/Pydantic 定义），节点间数据传递
- **Node**：执行单元（调用 LLM、工具、逻辑判断）
- **Edge**：节点连接（普通边、条件边 Command）
- **Checkpointer**：状态持久化（内存/MongoDB/Postgres）

**关键模式**：
- **Human-in-the-Loop**：`interrupt()` 暂停等待人工确认
- **循环与条件**：基于状态的条件路由实现循环 Agent
- **记忆持久化**：跨会话保持上下文（thread_id 管理）

```typescript
// 典型 State 定义
interface AgentState {
  messages: BaseMessage[];  // 对话历史
  next: string;             // 路由决策
  iteration: number;        // 循环计数
}

// 图构建流程
const builder = new StateGraph(AgentState);
builder.addNode("agent", callModel);
builder.addNode("tools", toolNode);
builder.addConditionalEdges("agent", shouldContinue, {
  continue: "tools",
  end: END
});
```

#### **阶段 4：生产化与生态集成（2-3 周）**

**目标**：部署生产环境，监控优化，完整工具链

### 五、学习建议与最佳实践

1. **优先掌握 LCEL**：新版 LangChain 已废弃 `LLMChain`，全面转向 LCEL 管道语法
2. **类型安全**：充分利用 TypeScript 类型推断，使用 Zod 定义 Tool 和 Output 结构
3. **模块化设计**：将 Prompt、Model、Parser 分离，便于测试和复用
4. **渐进式复杂度**：从简单 Chain → Memory Chain → RAG → Agent → LangGraph 逐步深入
5. **关注 Token 成本**：使用 `ConversationBufferWindowMemory` 或摘要策略控制上下文长度
