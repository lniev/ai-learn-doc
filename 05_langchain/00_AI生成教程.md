3. 核心模块详解

3.1 Model I/O 模块

核心概念

Model I/O是LangChain与LLM交互的基础层，包含三个核心组件：

- Models: 语言模型接口（LLM/ChatModel/Embedding）
- Prompts: 提示词管理与模板化
- Output Parsers: 结构化输出解析

代码示例：完整Model I/O流程

```python
# 安装依赖: pip install langchain langchain-openai pydantic

import os
from typing import List
from pydantic import BaseModel, Field
from langchain_openai import ChatOpenAI, OpenAIEmbeddings
from langchain_core.prompts import ChatPromptTemplate, FewShotPromptTemplate, PromptTemplate
from langchain_core.output_parsers import StrOutputParser, PydanticOutputParser, JsonOutputParser
from langchain_core.runnables import RunnablePassthrough

# 设置API Key
os.environ["OPENAI_API_KEY"] = "your-api-key"

# ==========================================
# 1. Prompt管理 - 基础模板
# ==========================================

# 简单PromptTemplate
simple_prompt = PromptTemplate.from_template(
    "请将以下中文翻译成英文：{text}"
)

# ChatPromptTemplate（推荐用于对话模型）
chat_prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一位专业的技术文档翻译专家，擅长将中文技术文档翻译为地道英文。"),
    ("human", "请翻译以下内容：{text}"),
    ("ai", "我会为您提供准确的技术翻译。"),
    ("human", "内容：{text}\n要求：保持技术术语准确，使用专业表达")
])

# FewShotPromptTemplate（少样本提示）
examples = [
    {"input": "你好", "output": "Hello"},
    {"input": "机器学习", "output": "Machine Learning"},
]
few_shot_prompt = FewShotPromptTemplate(
    examples=examples,
    example_prompt=PromptTemplate.from_template("Input: {input}\nOutput: {output}"),
    prefix="将中文翻译为英文：",
    suffix="Input: {input}\nOutput:",
    input_variables=["input"]
)

# ==========================================
# 2. 模型调用 - 使用LCEL（现代推荐方式）
# ==========================================

# 初始化模型
llm = ChatOpenAI(
    model="gpt-4o-mini",
    temperature=0.7,
    max_tokens=1000,
    timeout=30,
    max_retries=2
)

# LCEL链式调用（使用管道操作符 |）
# 这是LangChain现代推荐的核心语法[^9^]
translation_chain = (
    chat_prompt
    | llm
    | StrOutputParser()
)

# 执行
result = translation_chain.invoke({"text": "人工智能正在改变软件开发的方式。"})
print(f"翻译结果: {result}")

# ==========================================
# 3. 结构化输出解析 - Pydantic方式（生产推荐）
# ==========================================

class Task(BaseModel):
    """任务模型"""
    title: str = Field(description="任务标题")
    priority: int = Field(ge=1, le=5, description="优先级1-5")
    due_date: str = Field(description="截止日期(YYYY-MM-DD)")
    tags: List[str] = Field(default=[], description="任务标签")

# 创建结构化输出解析器
parser = PydanticOutputParser(pydantic_object=Task)

# 构建结构化输出链
structured_prompt = ChatPromptTemplate.from_messages([
    ("system", f"从用户输入中提取任务信息。\n{parser.get_format_instructions()}"),
    ("human", "{input_text}")
])

structured_chain = structured_prompt | llm | parser

# 测试结构化输出
task_input = "我需要在2024年12月25日前完成LangChain教程编写，优先级4，标签：AI、教育"
task = structured_chain.invoke({"input_text": task_input})
print(f"结构化结果: {task.model_dump()}")

# ==========================================
# 4. 高级：带Fallback的健壮链
# ==========================================

# 主要模型（GPT-4）带超时
primary_llm = ChatOpenAI(model="gpt-4o", temperature=0).with_config(
    {"run_name": "primary_llm", "timeout": 10}
)

# 备用模型（GPT-4-mini）
backup_llm = ChatOpenAI(model="gpt-4o-mini", temperature=0).with_config(
    {"run_name": "backup_llm"}
)

# 构建带Fallback的链[^9^]
robust_chain = (
    chat_prompt
    | primary_llm.with_fallbacks([backup_llm])
    | StrOutputParser()
)

# 异步批量处理（生产环境必备）
async def batch_translate(texts: List[str]):
    tasks = [translation_chain.ainvoke({"text": t}) for t in texts]
    return await asyncio.gather(*tasks)

# 使用示例
# import asyncio
# results = asyncio.run(batch_translate(["你好", "世界", "LangChain"]))
```

常见错误与解决方案

错误现象 原因 解决方案
`ValidationError` Pydantic模型字段不匹配 检查字段类型、必填项，使用`Field`添加描述
`TimeoutError` API响应超时 设置合理的`timeout`参数，使用`with_fallbacks`
`RateLimitError` 请求频率过高 实现指数退避重试，使用`max_retries`参数
输出格式不稳定 模型未遵循格式指令 使用`PydanticOutputParser`而非字符串解析

最佳实践建议

1. 始终使用LCEL语法: 管道操作符`|`比旧版`LLMChain`更清晰、可调试
2. 优先Pydantic结构化输出: 避免用正则解析自由文本，使用`with_retry()`自动修复格式错误
3. Prompt版本管理: 将Prompt存储为代码常量，使用Git管理版本，添加`run_name`和`tags`便于追踪
4. 异步优先: I/O密集型操作使用`ainvoke`/`abatch`，配合`asyncio.gather`提升吞吐量

3.2 Chains 模块

核心概念

Chains是LangChain的流程编排机制，将多个组件串联成工作流。现代LangChain推荐使用LCEL (LangChain Expression Language)而非旧版Chain类。

代码示例：从基础到复杂链

```python
from langchain_core.runnables import (
    RunnableLambda, RunnablePassthrough, RunnableParallel,
    RunnableBranch, chain
)
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI
from operator import itemgetter

# ==========================================
# 1. 基础链 - 简单顺序执行
# ==========================================

llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)

# 基础链：Prompt | LLM | Parser
basic_chain = (
    ChatPromptTemplate.from_template("总结以下内容：{text}")
    | llm
    | StrOutputParser()
)

# ==========================================
# 2. 复杂链 - 数据传递与转换
# ==========================================

# RunnablePassthrough：透传输入
# RunnableLambda：自定义函数
# itemgetter：提取字典字段

# 示例：多步骤处理链
multi_step_chain = (
    {
        "original": RunnablePassthrough(),
        "summary": basic_chain,
        "length": RunnableLambda(lambda x: len(x["text"]))
    }
    | RunnableLambda(lambda x: {
        "original": x["original"]["text"],
        "summary": x["summary"],
        "word_count": x["length"]
    })
)

# ==========================================
# 3. 并行链 - RunnableParallel
# ==========================================

# 同时执行多个独立任务
parallel_chain = RunnableParallel(
    sentiment=(
        ChatPromptTemplate.from_template("分析这段文本的情感（积极/消极/中性）：{text}")
        | llm
        | StrOutputParser()
    ),
    keywords=(
        ChatPromptTemplate.from_template("提取这段文本的3个关键词：{text}")
        | llm
        | StrOutputParser()
    ),
    summary=basic_chain  # 复用之前的链
)

# 使用
text = "LangChain让LLM应用开发变得简单高效，开发者可以快速构建AI应用"
result = parallel_chain.invoke({"text": text})
# 结果: {"sentiment": "积极", "keywords": "LangChain, LLM, 应用开发", "summary": "..."}

# ==========================================
# 4. 路由链 - RunnableBranch（条件分支）
# ==========================================

# 定义不同场景的Prompt
tech_prompt = ChatPromptTemplate.from_messages([
    ("system", "你是技术专家，用专业术语解释"),
    ("human", "{query}")
])
business_prompt = ChatPromptTemplate.from_messages([
    ("system", "你是商业顾问，关注ROI和市场影响"),
    ("human", "{query}")
])
general_prompt = ChatPromptTemplate.from_messages([
    ("system", "你是通用助手，提供清晰易懂的回答"),
    ("human", "{query}")
])

# 分类函数
def classify_query(query: str) -> str:
    tech_keywords = ["代码", "API", "架构", "算法", "Python"]
    business_keywords = ["成本", "收益", "市场", "客户", "盈利"]

    if any(k in query for k in tech_keywords):
        return "tech"
    elif any(k in query for k in business_keywords):
        return "business"
    return "general"

# 构建路由链
router_chain = (
    RunnablePassthrough()
    | RunnableLambda(lambda x: classify_query(x["query"]))
)

branch_chain = RunnableBranch(
    (lambda x: x["category"] == "tech", tech_prompt | llm | StrOutputParser()),
    (lambda x: x["category"] == "business", business_prompt | llm | StrOutputParser()),
    general_prompt | llm | StrOutputParser()  # 默认分支
)

full_chain = (
    {"query": RunnablePassthrough(), "category": router_chain}
    | branch_chain
)

# ==========================================
# 5. 自定义链 - 使用@chain装饰器（LangChain 0.1+）
# ==========================================

@chain
def custom_analysis_chain(inputs: dict) -> dict:
    """
    自定义分析链：先总结，再提取洞察，最后格式化输出
    """
    text = inputs["text"]
    analysis_type = inputs.get("type", "general")

    # 步骤1：生成摘要
    summary_prompt = ChatPromptTemplate.from_template(
        "用2句话总结：{text}"
    )
    summary = (summary_prompt | llm | StrOutputParser()).invoke({"text": text})

    # 步骤2：基于摘要生成洞察
    insight_prompt = ChatPromptTemplate.from_template(
        "基于以下摘要，提供3个关键洞察：\n{summary}\n\n洞察类型：{type}"
    )
    insights = (insight_prompt | llm | StrOutputParser()).invoke({
        "summary": summary,
        "type": analysis_type
    })

    return {
        "summary": summary,
        "insights": insights,
        "metadata": {
            "input_length": len(text),
            "analysis_type": analysis_type
        }
    }

# 使用自定义链
result = custom_analysis_chain.invoke({
    "text": "LangChain生态系统在2024年快速发展...",
    "type": "技术趋势"
})

# ==========================================
# 6. 高级：带状态管理的链（类似LangGraph轻量版）
# ==========================================

from typing import TypedDict, Annotated
from langchain_core.messages import AnyMessage
import operator

class ChainState(TypedDict):
    messages: Annotated[list[AnyMessage], operator.add]
    next_step: str
    iteration_count: int

def create_stateful_chain():
    """创建带迭代次数限制的状态链"""

    def step_1(state: ChainState) -> ChainState:
        if state["iteration_count"] > 3:
            return {**state, "next_step": "end"}
        # 处理逻辑
        return {**state, "iteration_count": state["iteration_count"] + 1}

    # 使用Runnable实现循环
    return RunnableLambda(step_1)

# ==========================================
# 7. 生产实践：链的调试与监控
# ==========================================

# 添加配置元数据（用于LangSmith追踪）
monitored_chain = basic_chain.with_config(
    {
        "run_name": "text_summarizer",
        "tags": ["production", "v1.0"],
        "metadata": {
            "model": "gpt-4o-mini",
            "temperature": 0,
            "use_case": "document_summary"
        }
    }
)

# 批量处理与错误处理
def safe_batch_invoke(chain, inputs, fallback_value="处理失败"):
    """带错误处理的批量调用"""
    results = []
    for inp in inputs:
        try:
            result = chain.invoke(inp)
            results.append(result)
        except Exception as e:
            print(f"处理失败 {inp}: {e}")
            results.append(fallback_value)
    return results
```

常见错误与解决方案

错误现象 原因 解决方案
`KeyError` in链执行 输入字典缺少必要字段 使用`RunnablePassthrough`确保字段传递，添加默认值
链执行顺序混乱 LCEL组合逻辑错误 使用括号明确分组，或分步测试子链
内存溢出（大批量） 同时加载过多数据 使用`abatch`替代`batch`，控制并发数
调试困难 链执行黑盒 启用LangSmith追踪，添加`run_name`和中间打印

最佳实践建议

1. 模块化设计: 将复杂链拆分为可复用的子链，每个子链单一职责
2. 类型安全: 使用TypedDict定义输入输出类型，配合Mypy静态检查
3. 尽早验证: 用`FakeListChatModel`进行单元测试，避免调用真实API
4. 流式支持: 对长输出启用`.stream()`，提升用户体验

3.3 RAG系统（Retrieval-Augmented Generation）

核心概念

RAG通过检索外部知识增强LLM回答能力，解决知识截止和幻觉问题。核心组件包括：

- Document Loaders: 加载各种格式文档
- Text Splitters: 智能分块保持语义
- Vector Stores: 存储文档向量
- Retrievers: 检索相关文档

代码示例：完整RAG系统实现

```python
# 安装依赖: pip install langchain langchain-openai langchain-community faiss-cpu pypdf

import os
from typing import List
from langchain_community.document_loaders import PyPDFLoader, TextLoader, WebBaseLoader
from langchain_text_splitters import (
    RecursiveCharacterTextSplitter,
    TokenTextSplitter,
    MarkdownHeaderTextSplitter
)
from langchain_openai import OpenAIEmbeddings
from langchain_community.vectorstores import FAISS, Chroma
from langchain_core.retrievers import BaseRetriever
from langchain_core.documents import Document
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.runnables import RunnablePassthrough
from langchain_openai import ChatOpenAI
from langchain_core.output_parsers import StrOutputParser

os.environ["OPENAI_API_KEY"] = "your-api-key"

# ==========================================
# 1. Document Loaders - 多源文档加载
# ==========================================

def load_documents(source_type: str, source_path: str) -> List[Document]:
    """
    统一文档加载接口
    """
    loaders = {
        "pdf": PyPDFLoader,
        "txt": TextLoader,
        "web": WebBaseLoader,
        "markdown": TextLoader  # Markdown用TextLoader后特殊处理
    }

    loader_class = loaders.get(source_type)
    if not loader_class:
        raise ValueError(f"不支持的文档类型: {source_type}")

    loader = loader_class(source_path)
    documents = loader.load()

    # 添加元数据
    for doc in documents:
        doc.metadata.update({
            "source_type": source_type,
            "source_path": source_path,
            "load_timestamp": str(datetime.now())
        })

    return documents

# 示例：加载PDF
# docs = load_documents("pdf", "./data/annual_report.pdf")

# ==========================================
# 2. Text Splitters - 智能文本分割
# ==========================================

def create_smart_splitter(doc_type: str = "general"):
    """
    根据文档类型选择最佳分割策略[^9^]
    """
    splitters = {
        "general": RecursiveCharacterTextSplitter(
            chunk_size=500,
            chunk_overlap=100,
            length_function=len,
            separators=["\n\n", "\n", "。", "，", " ", ""]
        ),
        "code": RecursiveCharacterTextSplitter(
            chunk_size=1000,
            chunk_overlap=200,
            separators=["\nclass ", "\ndef ", "\n\n", "\n", " ", ""]
        ),
        "markdown": MarkdownHeaderTextSplitter(
            headers_to_split_on=[("#", "Header 1"), ("##", "Header 2")]
        ),
        "token": TokenTextSplitter(
            chunk_size=500,
            chunk_overlap=50,
            encoding_name="cl100k_base"  # OpenAI编码
        )
    }
    return splitters.get(doc_type, splitters["general"])

# 高级分割：语义保持
def semantic_chunking(documents: List[Document], target_chunk_size: int = 500):
    """
    基于语义的智能分块：在段落边界处分割，避免切断语义单元
    """
    splitter = RecursiveCharacterTextSplitter(
        chunk_size=target_chunk_size,
        chunk_overlap=target_chunk_size * 0.2,  # 20%重叠保持上下文
        separators=["\n\n", "\n", ". ", "。", "；", "，", " ", ""],
        length_function=len
    )

    chunks = splitter.split_documents(documents)

    # 添加chunk编号和上下文信息
    for i, chunk in enumerate(chunks):
        chunk.metadata.update({
            "chunk_index": i,
            "chunk_total": len(chunks),
            "prev_chunk_id": i - 1 if i > 0 else None,
            "next_chunk_id": i + 1 if i < len(chunks) - 1 else None
        })

    return chunks

# 3. Vector Stores - 向量存储与检索
# ==========================================

class VectorStoreManager:
    """
    向量存储管理器：封装FAISS/Chroma操作
    """

    def __init__(self, embedding_model=None):
        self.embeddings = embedding_model or OpenAIEmbeddings(
            model="text-embedding-3-small"
        )
        self.vectorstore = None

    def create_from_documents(self, documents: List[Document], store_type: str = "faiss"):
        """从文档创建向量存储"""
        if store_type == "faiss":
            self.vectorstore = FAISS.from_documents(documents, self.embeddings)
        elif store_type == "chroma":
            self.vectorstore = Chroma.from_documents(documents, self.embeddings)
        else:
            raise ValueError(f"不支持的存储类型: {store_type}")
        return self

    def save_local(self, path: str):
        """持久化存储（仅FAISS支持）"""
        if isinstance(self.vectorstore, FAISS):
            self.vectorstore.save_local(path)
        else:
            raise NotImplementedError("Chroma持久化请使用Chroma原生API")

    def load_local(self, path: str):
        """加载本地存储"""
        self.vectorstore = FAISS.load_local(
            path, self.embeddings, allow_dangerous_deserialization=True
        )
        return self

    def get_retriever(self, search_type: str = "mmr", **kwargs):
        """
        获取检索器

        search_type:
        - "similarity": 相似度检索（默认）
        - "mmr": 最大边际相关性，增加多样性[^9^]
        - "similarity_score_threshold": 带分数阈值
        """
        search_kwargs = {
            "k": kwargs.get("k", 4),
            "fetch_k": kwargs.get("fetch_k", 20),  # MMR候选池
            "lambda_mult": kwargs.get("lambda_mult", 0.5)  # MMR多样性参数
        }

        if search_type == "similarity_score_threshold":
            search_kwargs["score_threshold"] = kwargs.get("score_threshold", 0.7)

        return self.vectorstore.as_retriever(
            search_type=search_type,
            search_kwargs=search_kwargs
        )

# 4. 高级Retrievers - 多策略检索
# ==========================================

class MultiStrategyRetriever(BaseRetriever):
    """
    多策略检索器：结合多种检索方法提升召回率
    """

    def __init__(self, base_retriever: BaseRetriever, llm=None):
        super().__init__()
        self.base_retriever = base_retriever
        self.llm = llm or ChatOpenAI(temperature=0)

    def _get_relevant_documents(self, query: str) -> List[Document]:
        # 策略1：原始查询检索
        docs1 = self.base_retriever.get_relevant_documents(query)

        # 策略2：查询扩展（使用LLM生成同义查询）
        expansion_prompt = ChatPromptTemplate.from_template(
            "为以下查询生成3个同义或相关的搜索查询，每行一个：\n查询：{query}"
        )
        expansion_chain = expansion_prompt | self.llm | StrOutputParser()
        expanded_queries = expansion_chain.invoke({"query": query}).strip().split("\n")

        # 检索扩展查询
        all_docs = docs1.copy()
        for eq in expanded_queries[:3]:  # 限制扩展数量
            if eq.strip():
                docs = self.base_retriever.get_relevant_documents(eq.strip())
                all_docs.extend(docs)

        # 去重（基于内容哈希）
        seen = set()
        unique_docs = []
        for doc in all_docs:
            content_hash = hash(doc.page_content[:100])
            if content_hash not in seen:
                seen.add(content_hash)
                unique_docs.append(doc)

        return unique_docs[:6]  # 返回前6个不重复结果

# 5. 完整RAG Chain - 端到端实现
# ==========================================

def create_rag_chain(vectorstore_manager: VectorStoreManager):
    """
    创建生产级RAG链[^9^]
    """

    # 使用MMR检索器增加多样性
    retriever = vectorstore_manager.get_retriever(
        search_type="mmr",
        k=4,
        fetch_k=20,
        lambda_mult=0.5
    )

    # RAG Prompt模板（强制引用来源）
    RAG_PROMPT = ChatPromptTemplate.from_messages([
        ("system", """你是专业的问答助手。请严格根据提供的参考资料回答问题。
如果在参考资料中找不到答案，请明确说"根据现有资料无法回答"。
引用来源时使用[1], [2]等标记。"""),
        ("human", """问题：{question}

参考资料：
{context}

请提供详细回答，并标注信息来源：""")
    ])

    # 格式化文档函数（带引用标记）
    def format_docs_with_citation(docs: List[Document]) -> str:
        formatted = []
        for i, doc in enumerate(docs, 1):
            source = doc.metadata.get("source", "未知来源")
            formatted.append(f"[{i}] 来源：{source}\n内容：{doc.page_content}")
        return "\n\n".join(formatted)

    # 构建RAG链
    rag_chain = (
        {
            "context": retriever | format_docs_with_citation,
            "question": RunnablePassthrough()
        }
        | RAG_PROMPT
        | ChatOpenAI(model="gpt-4o", temperature=0)
        | StrOutputParser()
    )

    return rag_chain
```

常见错误与解决方案

错误现象 原因 解决方案
检索结果不相关 Chunk过小或Embedding质量差 增大chunk_size至500-1000，使用`text-embedding-3`系列模型
回答出现幻觉 未强制模型基于上下文 Prompt中明确指令"仅根据资料回答"，添加验证步骤
长文档处理慢 未使用异步或批处理 使用`aload()`异步加载，`abatch`批量处理
向量索引过大 未优化存储 使用量化索引（FAISS IVF），或切换到Chroma/Pinecone

最佳实践建议

1. Chunk策略: 技术文档用较大chunk（1000+），FAQ用较小chunk（300-500），保持20%重叠
2. MMR检索: 默认使用MMR替代简单相似度，设置`fetch_k=20, k=4`平衡多样性与相关性
3. 引用强制: Prompt要求模型标注来源，后端验证回答中是否包含引用标记
4. 混合检索: 结合关键词检索（BM25）和向量检索，使用`EnsembleRetriever`

核心概念

```python
class CustomAgentExecutor:
    """
    自定义Agent执行器：添加前置/后置处理、人工审核点
    """

    def __init__(self, agent_executor: AgentExecutor):
        self.agent_executor = agent_executor
        self.sensitive_tools = ["delete", "update", "transfer"]  # 敏感操作

    def invoke(self, input_data: dict):
        # 前置检查
        query = input_data.get("input", "")
        if self._is_sensitive(query):
            print("⚠️ 检测到敏感操作请求，需要人工确认")
            # 这里可以接入人工审核流程
            if not self._human_approval(query):
                return {"output": "操作已取消（未通过安全审核）"}

        # 执行
        result = self.agent_executor.invoke(input_data)

        # 后置处理：记录日志
        self._log_execution(input_data, result)

        return result

    def _is_sensitive(self, query: str) -> bool:
        return any(tool in query.lower() for tool in self.sensitive_tools)

    def _human_approval(self, query: str) -> bool:
        # 模拟人工审核
        response = input(f"确认执行 '{query}'? (y/n): ")
        return response.lower() == 'y'

    def _log_execution(self, input_data: dict, result: dict):
        print(f"[LOG] Query: {input_data['input'][:50]}... Result: {result['output'][:50]}...")
```

常见错误与解决方案

错误现象 原因 解决方案
`Agent stopped due to max iterations` 任务过复杂或Agent陷入循环 增加`max_iterations`，简化任务，添加更清晰的工具描述
`Could not parse LLM output` LLM输出格式不符合预期 启用`handle_parsing_errors=True`，使用Tool Calling Agent替代ReAct

无限循环 Agent在相同步骤重复 添加记忆组件，限制工具调用次数，使用Plan-and-Execute

最佳实践建议

1. 工具描述优化: 工具描述应包含"何时使用"和"输入格式示例"，避免歧义
2. 权限控制: 敏感工具（删除、转账）添加人工审核节点，使用`CustomAgentExecutor`模式
3. 降级策略: 复杂任务使用Plan-and-Execute，简单任务使用Tool Calling，避免过度设计
4. 监控必备: 使用LangSmith追踪每次工具调用和决策过程，便于调试优化

```python
class LongTermMemory:
    """
    基于向量存储的长期记忆系统
    可以跨会话记住用户偏好、事实等信息
    """

    def __init__(self):
        self.embeddings = OpenAIEmbeddings()
        self.llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)
        self.vectorstore = None
        self.memory_retriever = None

    def initialize_from_facts(self, facts: List[str]):
        """
        从事实列表初始化长期记忆
        """
        documents = [
            Document(
                page_content=fact,
                metadata={
                    "timestamp": str(datetime.now()),
                    "type": "user_fact",
                    "id": i
                }
            )
            for i, fact in enumerate(facts)
        ]

        self.vectorstore = FAISS.from_documents(documents, self.embeddings)
        self.memory_retriever = self.vectorstore.as_retriever(
            search_kwargs={"k": 3}
        )

        return self

    def add_fact(self, fact: str, fact_type: str = "general"):
        """动态添加新事实"""
        if self.vectorstore is None:
            self.initialize_from_facts([fact])
        else:
            doc = Document(
                page_content=fact,
                metadata={
                    "timestamp": str(datetime.now()),
                    "type": fact_type
                }
            )
            self.vectorstore.add_documents([doc])

    def create_memory_augmented_chain(self):
        """
        创建增强记忆的对话链
        """
        # VectorStoreRetrieverMemory
        vector_memory = VectorStoreRetrieverMemory(
            retriever=self.memory_retriever,
            memory_key="long_term_memory",
            input_key="input"
        )

        # 结合短期Buffer记忆
        buffer_memory = ConversationBufferMemory(
            memory_key="short_term_memory",
            return_messages=True
        )

        prompt = ChatPromptTemplate.from_messages([
            ("system", """你是智能助手。回答时参考：
1. 长期记忆（用户事实）：{long_term_memory}
2. 近期对话：{short_term_memory}"""),
            MessagesPlaceholder(variable_name="short_term_memory"),
            ("human", "{input}")
        ])

        # 手动组合记忆（因为LangChain原生不支持多记忆源）
        def combine_memories(input_data: dict) -> dict:
            long_term = vector_memory.load_memory_variables(input_data)
            short_term = buffer_memory.load_memory_variables(input_data)

            return {
                "input": input_data["input"],
                "long_term_memory": long_term["long_term_memory"],
                "short_term_memory": short_term["short_term_memory"]
            }

        chain = (
            RunnableLambda(combine_memories)
            | prompt
            | self.llm
        )

        return chain, buffer_memory  # 返回buffer_memory以便保存上下文

class SmartMemory:
    """
    智能记忆系统：自动提取关键信息，选择性记忆
    """

    def __init__(self):
        self.llm = ChatOpenAI(model="gpt-4o", temperature=0)
        self.important_facts: List[Dict[str, Any]] = []
        self.conversation_buffer: List[Dict] = []

    def extract_facts(self, human_msg: str, ai_msg: str) -> List[str]:
        """
        使用LLM从对话中提取值得长期记忆的事实
        """
        extraction_prompt = ChatPromptTemplate.from_messages([
            ("system", "分析以下对话，提取关于用户的关键事实（偏好、身份、目标等）。"
                      "只返回事实列表，每行一个，没有则返回'无'。"),
            ("human", "用户：{human_msg}\n助手：{ai_msg}")
        ])

        chain = extraction_prompt | self.llm | StrOutputParser()
        result = chain.invoke({
            "human_msg": human_msg,
            "ai_msg": ai_msg
        })

        if result.strip() == "无":
            return []
        return [f.strip() for f in result.split("\n") if f.strip()]

    def save_interaction(self, human_msg: str, ai_msg: str):
        """
        保存交互，自动提取重要事实
        """
        # 保存到短期缓冲
        self.conversation_buffer.append({
            "human": human_msg,
            "ai": ai_msg,
            "timestamp": datetime.now()
        })

        # 提取重要事实
        facts = self.extract_facts(human_msg, ai_msg)
        for fact in facts:
            self.important_facts.append({
                "fact": fact,
                "extracted_at": datetime.now(),
                "source": "conversation"
            })
            print(f"  [记忆提取] {fact}")

    def get_context(self, query: str, short_term_limit: int = 3) -> Dict:
        """
        获取上下文：短期对话 + 相关长期事实
        """
        # 短期记忆
        short_term = self.conversation_buffer[-short_term_limit:]

        # 长期记忆：简单关键词匹配（生产环境可用向量检索）
        relevant_facts = [
            f for f in self.important_facts
            if any(word in f["fact"].lower() for word in query.lower().split())
        ]

        return {
            "short_term": short_term,
            "long_term": relevant_facts[:3]  # 限制数量
        }

class MemoryOptimizer:
    """
    记忆成本优化：Token管理与智能压缩[^3^]
    """

    def __init__(self):
        self.llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)

    def compress_history(self, messages: List[Any], target_tokens: int = 1000) -> str:
        """
        压缩长历史为摘要
        """
        # 估算当前Token数（简化计算，实际应使用tiktoken）
        current_tokens = sum(len(str(m.content)) for m in messages) / 4

        if current_tokens <= target_tokens:
            return "\n".join([f"{m.type}: {m.content}" for m in messages])

        # 需要压缩：分段总结
        summary_prompt = ChatPromptTemplate.from_template(
            "将以下对话历史总结为关键要点（保留重要事实和决策）：\n\n{history}"
        )

        history_text = "\n".join([f"{m.type}: {m.content}" for m in messages])
        chain = summary_prompt | self.llm | StrOutputParser()

        return chain.invoke({"history": history_text})

    def sliding_window_with_summary(self, messages: List[Any], window_size: int = 4):
        """
        滑动窗口+摘要：最近window_size轮详细，更早的总结
        """
        if len(messages) <= window_size:
            return messages

        recent = messages[-window_size:]
        older = messages[:-window_size]

        # 总结早期对话
        older_summary = self.compress_history(older, target_tokens=500)

        # 组合：系统消息（总结）+ 近期详细消息
        combined = [
            SystemMessage(content=f"早期对话总结：{older_summary}")
        ] + recent

        return combined
```

常见错误与解决方案

错误现象 原因 解决方案
`Token limit exceeded` 记忆内容过长 使用`ConversationSummaryMemory`或滑动窗口，设置`max_token_limit`
记忆未生效 未正确传递memory_key 检查Prompt中的变量名与`memory_key`一致，使用`return_messages=True`
跨会话记忆丢失 未持久化 使用`FileChatMessageHistory`或数据库存储，定期保存向量索引
长期记忆检索不相关 Embedding质量问题 使用更好的Embedding模型，添加元数据过滤，调整`search_kwargs`

最佳实践建议

1. 分层记忆: 短期用BufferWindow（k=3-5），长期用VectorStore，关键事实用数据库
2. Token预算管理: 为记忆预留30%上下文窗口，超过则触发总结或压缩
3. 隐私保护: 敏感信息（密码、个人信息）使用`RedactingMemory`过滤或加密存储
4. 记忆分离: 区分"对话历史"（短期）和"用户画像"（长期），避免混淆

LangChain作为LLM应用开发的主流框架，核心价值在于组件化和可组合性。学习时应：

1. 先掌握LCEL语法：管道操作符`|`是现代LangChain的灵魂

4. 调试能力：善用`verbose=True`和LangSmith，理解执行流程比写代码更重要

建议学习总投入2-3个月（每天2-3小时），按阶段完成三个实战项目，即可具备独立开发企业级LLM应用的能力。
