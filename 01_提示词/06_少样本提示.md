## 6. 少样本提示（Few-Shot Prompting）

### 6.1 核心概念

| 技术             | 定义                                     | 与零样本区别                     |
| ---------------- | ---------------------------------------- | -------------------------------- |
| **少样本提示**   | 提供 1-5 个输入-输出示例，让模型模仿模式 | 用示例教"做什么"，而非仅描述任务 |
| **少样本思维链** | 示例中包含完整推理过程，模型模仿推理模式 | 教"怎么做"，而非仅教"答案是什么" |

---

### 6.2 少样本提示（标准版）

#### 基本结构

```markdown
【任务描述】（可选）
【示例1】输入 → 输出
【示例2】输入 → 输出
【示例3】输入 → 输出
【待解决问题】输入 →
```

#### 完整示例：情感分类

```markdown
判断评论情感倾向（正面/负面/中性）。

评论：这部电影太棒了，演技炸裂！
情感：正面

评论：排队两小时，味道一般般。
情感：负面

评论：新品上市，价格待定。
情感：中性

评论：服务员态度恶劣，再也不来了。
情感：
```

#### 关键设计原则

| 原则           | 说明                           | 反例                                 |
| -------------- | ------------------------------ | ------------------------------------ |
| **示例代表性** | 覆盖主要变体和边界情况         | 3个示例全是正面，模型学会忽略负面    |
| **格式一致性** | 输入输出结构严格统一           | 示例1用JSON，示例2用纯文本，模型混乱 |
| **数量适中**   | 通常 3-5 个，复杂任务可到 8-10 | 示例过多导致上下文窗口拥挤           |
| **顺序影响**   | 最后示例影响最大（近因效应）   | 关键类别放最后示例                   |
| **标签平衡**   | 分类任务中各类别示例均衡       | 10个示例9个A类，模型偏预测A          |
