## 1.角色扮演

- 你现在是xxx，你要xx，到达xxx + 具体限制

## 2.提示词核心技巧

| 元素 | 描述                                             | 例子                                         | 备注     |
| ---- | ------------------------------------------------ | -------------------------------------------- | -------- |
| 指令 | 想要执行的特定任务                               | 生成代码，翻译，解释，解析，分析             |          |
| 场景 | 外部信息补充的信息告诉大模型更改的执行           | 你是一个前端开发人员，熟练使用react          | 角色扮演 |

## 3.参数微调

### 3.1温度

#### 🎯 推荐测试 Temperature 梯度

| 档位     | 数值 | 预期特征                                             | 适用场景                             |
| -------- | ---- | ---------------------------------------------------- | ------------------------------------ |
| **冻结** | 0.0  | 完全按照要求执行，随机变化小，比较死板               | 数据的指定输出 ，实时查询，代码生成  |
| **保守** | 0.3  | 有点随机，但任然不够                                 | 写技术文档，分析任务等               |
| **平衡** | 0.7  | 均衡                                                 | 日常对话，通用助手                   |
| **活跃** | 1.0  | 有一定随机性，但不会太偏离要求                       | 营销文案，头脑风暴需要一些创意性内容 |
| **奔放** | 1.3  | 高度发散，可能会失去约束。                           | 创意写作，艺术类                     |
| **混沌** | 1.6+ | 不太可控，随机性强，可能出现逻辑断裂，生成内容不可控 | 实验性创作                           |

> 建议重点测试 **0.3、0.7、1.0、1.3** 四个关键节点，能清晰观察从保守到创意的渐变。

### 3.2 Top-P

#### 3.2.1🎯 Top-P 核心机制

| 参数                    | 机制                  | 效果                     |
| ----------------------- | --------------------- | ------------------------ |
| **低 Top-P (0.1-0.5)**  | 仅采样高概率核心token | 输出保守、确定、事实性强 |
| **中 Top-P (0.7-0.9)**  | 平衡质量与多样性      | 通用场景，兼顾准确与流畅 |
| **高 Top-P (0.95-1.0)** | 纳入长尾低概率token   | 创意丰富，但可能引入噪声 |

> Top-P 与 Top-K 区别：Top-P 是动态筛选（按概率累积），Top-K 是固定数量筛选

---

#### 3.2.2📊 场景化 Top-P 配置

##### 1. 精确输出场景

| 参数  | 推荐值                       | 配合设置            |
| ----- | ---------------------------- | ------------------- |
| Top-P | **0.1-0.3**                  | Temperature 0.0-0.2 |
| 适用  | 数学计算、代码生成、事实问答 |                     |
| 效果  | 几乎贪婪解码，错误率最低     |                     |

**示例**：SQL生成、JSON结构化输出、数值计算

---

##### 2. 平衡场景（默认）

| 参数  | 推荐值                   | 配合设置            |
| ----- | ------------------------ | ------------------- |
| Top-P | **0.85-0.9**             | Temperature 0.6-0.8 |
| 适用  | 通用对话、客服、内容改写 |                     |
| 效果  | 自然流畅，偶尔有亮点     |                     |

**OpenAI/DeepSeek默认**：0.9 [^32^]

---

##### 3. 创意发散场景

| 参数  | 推荐值                   | 配合设置            |
| ----- | ------------------------ | ------------------- |
| Top-P | **0.95-1.0**             | Temperature 0.8-1.2 |
| 适用  | 头脑风暴、诗歌、营销文案 |                     |
| 效果  | 词汇丰富，意象跳跃       |                     |

> 注意：Top-P=1.0 时效果类似纯采样，需配合 Temperature 控制

---

#### 3.2.3⚡ Top-P vs Temperature 联动策略

| 策略         | Temperature | Top-P | 适用场景               |
| ------------ | ----------- | ----- | ---------------------- |
| **严格精确** | 0.0         | 0.1   | 代码、数学、工具调用   |
| **保守可靠** | 0.3         | 0.7   | 技术文档、分析报告     |
| **标准通用** | 0.7         | 0.9   | 日常对话、问答         |
| **创意探索** | 1.0         | 0.95  | 写作、命名、策划       |
| **极端实验** | 1.3         | 1.0   | 艺术生成（质量不稳定） |

> **黄金法则**：两者同向调整（都高或都低），避免一高一低冲突 [^32^]

---

### 3.3.1 top_p 与 temperature 差异

深层差异：何时用哪个？

##### 用 Temperature 当：

- 需要**全局控制"冒险程度"**
- 想让模型"更自信"或"更开放"
- 调整**概率曲线的形状**

##### 用 Top-P 当：

- 需要**硬性过滤垃圾选项**
- 想保留**动态数量的优质候选**
- 防止**绝对不可能的词**出现

#### 协同工作原理

Temperature 决定"敢不敢选次优" Top-P 决定"有哪些选项可以选"

动态调整公式：

- 创意需求低 → Top-P = 0.5 + (Temperature × 0.3)
- 创意需求高 → Top-P = 0.9 + (Temperature × 0.08)

### 3.3.2 Top-K

#### 工作原理

模型输出词汇按概率排序，保留Top-K个词汇

#### 与 Top-P 的本质区别

| 特性       | Top-K                | Top-P              |
| ---------- | -------------------- | ------------------ |
| 筛选方式   | **固定数量**         | **动态概率累积**   |
| 候选池大小 | 恒定K个              | 随置信度变化       |
| 上下文适应 | 死板                 | 灵活               |
| 计算速度   | 更快（固定排序截断） | 稍慢（需累积计算） |
| 极端情况   | K=1=贪婪，K大=纯采样 | P=0=空，P=1=纯采样 |

---

#### 📊 Top-K 档位配置

| 档位     | 数值    | 效果                     | 适用场景               |
| -------- | ------- | ------------------------ | ---------------------- |
| **贪婪** | 1       | 只选最高概率词，完全确定 | 数学计算、代码语法     |
| **严格** | 3-5     | 极小选择空间，高度保守   | 事实问答、JSON输出     |
| **保守** | 10-20   | 有限选项，质量优先       | 技术文档、摘要         |
| **标准** | 40-50   | 平衡质量与多样性         | 通用对话（**默认值**） |
| **宽松** | 100+    | 大量候选，创意增强       | 头脑风暴、写作         |
| **禁用** | 0或不设 | 所有词汇参与             | 纯采样实验             |

> 现代模型（GPT-4/Claude/DeepSeek）默认 **Top-K=40~50** [^33^]

---

#### ⚔️ Top-K vs Temperature 联动

| 组合                            | 效果                   | 风险                |
| ------------------------------- | ---------------------- | ------------------- |
| **低K + 低Temp** (K=5, T=0.3)   | 极度确定，几乎唯一输出 | 重复、死板          |
| **低K + 高Temp** (K=5, T=1.5)   | 小池内疯狂随机         | 逻辑断裂            |
| **高K + 低Temp** (K=100, T=0.3) | 候选多但只选头部       | 浪费算力，效果同低K |
| **高K + 高Temp** (K=100, T=1.2) | 大池内充分探索         | 质量不稳定          |

> **推荐策略**：固定K=40~50，用Temperature调节风格，避免双高冲突

---

#### 🎯 场景化配置方案

##### 1. 确定性任务

```python
Top-K = 1          # 贪婪解码
# 或
Top-K = 3-5        # 极小安全池
Temperature = 0.0-0.3
```

##### 2. 高质量内容

```
Top-K = 20-40      # 过滤绝对垃圾
Temperature = 0.5-0.7
```

新闻摘要、产品描述、客服回复

##### 3. 创意探索

```
Top-K = 50-100     # 保留长尾惊喜
Temperature = 0.8-1.0
# 配合
Top-P = 0.9        # 双重保险，防极端异常

```

适用：品牌命名、广告文案、故事开头

##### 4.问题诊断

| 现象         | 原因         | 修复                   |
| ------------ | ------------ | ---------------------- |
| 输出重复单调 | K过小        | 提至40+                |
| 出现生僻怪词 | K过大+Temp高 | K降至20，或加Top-P=0.9 |
| 前后逻辑跳跃 | K大+Temp双高 | 固定K=50，单降Temp     |
| 算力消耗高   | K过大        | 降至50足够，收益递减   |

### 3.4 Top-K 与 Top-P 与 Temperature 联动

- **顺序结论**：Top-K 先硬截断 → Top-P 后软筛选 -> Temperature 最后调随机
- K 和 P 都是在预测每一个 token（字/词）时实时生效
- 先 K 砍数量，后 P 选质量，最后 Temp 调随机
