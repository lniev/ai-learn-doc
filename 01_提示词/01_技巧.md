## 1.角色扮演

- 你现在是xxx，你要xx，到达xxx + 具体限制

## 2.提示词核心技巧

| 元素 | 描述                                             | 例子                                         | 备注     |
| ---- | ------------------------------------------------ | -------------------------------------------- | -------- |
| 指令 | 想要执行的特定任务                               | 生成代码，翻译，解释，解析，分析             |          |
| 场景 | 外部信息补充的信息告诉大模型更改的执行           | 你是一个前端开发人员，熟练使用react          | 角色扮演 |
| 输入 | 输入内容或问题或者上下文，模型需要接收处理的东西 | 需要推理的问题                               |          |
| 输出 | 指定的输出格式或者类型                           | 参考xx代码一样，生成列表代码 1000个字 md格式 |          |

## 3. 零样本提示词（Zero-Shot Prompting）

### 3.1 基本形式

直接指令，无示例

**示例：**

- 例1: "将这段中文翻译成英文。"
- 例2: "总结以下文章的主要观点。"
- 例3: "判断这条评论的情感倾向。"

### 3.2 优化技巧

| 技巧           | 示例                                      | 效果         |
| -------------- | ----------------------------------------- | ------------ |
| **明确角色**   | "作为资深律师，分析这份合同的风险点"      | 激活领域知识 |
| **指定格式**   | "用JSON返回，包含risk_level和description" | 结构化输出   |
| **限定长度**   | "用100字以内解释量子纠缠"                 | 控制输出粒度 |
| **加约束条件** | "解释给10岁孩子听，避免专业术语"          | 调整表达风格 |

### 3.3 完整优化模板

```markdown
【角色】你是一位经验丰富的数据分析师
【任务】分析Q3销售数据下滑的原因
【格式】分三点阐述，每点包含"现象-根因-建议"
【约束】避免使用技术术语，面向非财务背景的CEO
【输入】{{data}}
```

## 4. 零样本思维链（Zero-Shot CoT）

### 4.1 核心发现

> **Kojima 等人 2022 年论文**：只需加一句 "Let's think step by step"，模型自动展开推理链，数学题准确率从 **17.7%** 提升到 **78.7%**

### 4.2 标准触发短语

| 短语                                    | 适用场景           | 效果强度   |
| --------------------------------------- | ------------------ | ---------- |
| "Let's think step by step."             | 英文数学题、逻辑题 | ⭐⭐⭐⭐⭐ |
| "请逐步思考。"                          | 中文推理任务       | ⭐⭐⭐⭐⭐ |
| "Let's work through this step by step." | 代码调试、复杂分析 | ⭐⭐⭐⭐   |
| "请详细展示你的推理过程。"              | 需要可视化解题路径 | ⭐⭐⭐⭐   |
| "先分析，再结论。"                      | 防止模型跳步       | ⭐⭐⭐     |

### 4.3 效果对比示例

**问题：**

> 一个农场有鸡和兔共35只，脚共94只。鸡兔各几只？

**标准零样本（可能跳步/错误）：**

```
鸡23只，兔12只。  ← 可能直接给答案，或计算错误
```

**零样本 CoT（强制展开推理）：**

```
设鸡有x只，兔有y只。
第一步：根据头数列方程 x + y = 35
第二步：根据脚数列方程 2x + 4y = 94
第三步：化简得 x + 2y = 47
第四步：减去第一式得 y = 12
第五步：代入得 x = 23
结论：鸡23只，兔12只。  ← 可验证，易纠错
```

### 4.4 进阶技巧：结构化 CoT

#### 技巧1：强制分步格式

```markdown
请按以下步骤解答：

1. 【理解】重述题目条件和目标
2. 【建模】列出关键变量和关系
3. 【求解】逐步计算，展示每步结果
4. 【验证】检查结果是否符合所有条件
5. 【结论】给出最终答案
```

#### 技巧2：自我验证触发

```markdown
解答后，请：

- 用另一种方法交叉验证
- 检查边界条件是否满足
- 确认答案合理性
```

#### 技巧3：多路径探索

```markdown
请提供两种不同思路的解法，比较优劣。
```

### 4.5 适用场景决策树

```
问题类型判断
    │
    ├─ 事实检索（常识/定义）→ 标准零样本
    │   例："法国首都是哪？"
    │
    ├─ 简单变换（翻译/格式化）→ 标准零样本
    │   例："将CSV转JSON"
    │
    ├─ 单步推理（简单计算）→ 标准零样本 或 轻量CoT
    │   例："100的平方根是多少？"
    │
    └─ 多步推理（数学/逻辑/决策）→ 必须用零样本CoT
        例：鸡兔同笼、代码调试、商业分析、医疗诊断
```

### 4.6 关键洞察

| 要点                      | 说明                                     |
| ------------------------- | ---------------------------------------- |
| **零样本 CoT 是免费午餐** | 不加示例、不微调，只改提示词，显著提效   |
| **大模型收益更大**        | GPT-3.5/4、Claude 3、DeepSeek 等效果最佳 |
| **简单任务可能过度**      | "2+2=?" 用 CoT 反而冗长，标准零样本更快  |
| **可与其他技术叠加**      | Zero-Shot CoT + 角色设定 + 输出格式约束  |
| **非推理模型效果差**      | 纯生成模型（早期GPT-3）可能产生假推理    |

## 5. 技术对比总结

| 技术             | 英文                             | 核心机制                   | 适用场景                   |
| ---------------- | -------------------------------- | -------------------------- | -------------------------- |
| **零样本提示**   | Zero-Shot Prompting              | 直接提问，不给示例         | 模型已具备该能力，简单任务 |
| **零样本思维链** | Zero-Shot Chain-of-Thought (CoT) | 加一句"逐步思考"，激活推理 | 需要多步推理的复杂问题     |

---

> 🔥 **一句话总结**
>
> **标准零样本** = "直接答"
> **零样本 CoT** = "请展示思考过程再答"
>
> 后者用 4 个单词（Let's think step by step）解锁模型隐藏推理能力，是提示工程史上最高性价比的技巧之一。

## 6. 少样本提示（Few-Shot Prompting）

### 6.1 核心概念

| 技术             | 定义                                     | 与零样本区别                     |
| ---------------- | ---------------------------------------- | -------------------------------- |
| **少样本提示**   | 提供 1-5 个输入-输出示例，让模型模仿模式 | 用示例教"做什么"，而非仅描述任务 |
| **少样本思维链** | 示例中包含完整推理过程，模型模仿推理模式 | 教"怎么做"，而非仅教"答案是什么" |

---

### 6.2 少样本提示（标准版）

#### 基本结构

```markdown
【任务描述】（可选）
【示例1】输入 → 输出
【示例2】输入 → 输出
【示例3】输入 → 输出
【待解决问题】输入 →
```

#### 完整示例：情感分类

```markdown
判断评论情感倾向（正面/负面/中性）。

评论：这部电影太棒了，演技炸裂！
情感：正面

评论：排队两小时，味道一般般。
情感：负面

评论：新品上市，价格待定。
情感：中性

评论：服务员态度恶劣，再也不来了。
情感：
```

#### 关键设计原则

| 原则           | 说明                           | 反例                                 |
| -------------- | ------------------------------ | ------------------------------------ |
| **示例代表性** | 覆盖主要变体和边界情况         | 3个示例全是正面，模型学会忽略负面    |
| **格式一致性** | 输入输出结构严格统一           | 示例1用JSON，示例2用纯文本，模型混乱 |
| **数量适中**   | 通常 3-5 个，复杂任务可到 8-10 | 示例过多导致上下文窗口拥挤           |
| **顺序影响**   | 最后示例影响最大（近因效应）   | 关键类别放最后示例                   |
| **标签平衡**   | 分类任务中各类别示例均衡       | 10个示例9个A类，模型偏预测A          |

## 7. 少样本思维链（Few-Shot CoT）

### 7.1 核心区别：示例内含推理过程

```
标准少样本：   问题 → 答案
少样本 CoT：   问题 → 推理步骤 → 答案
```

### 7.2 完整示例：数学推理

```markdown
请解答数学应用题，展示推理过程。

【例1】
问题：小明有5个苹果，给了小红2个，又买了3个，现在有几个？
推理：

- 初始：5个
- 给小红后：5 - 2 = 3个
- 又买3个后：3 + 3 = 6个
  答案：6个

【例2】
问题：火车时速60公里，行驶2.5小时，路程多少？
推理：

- 公式：路程 = 速度 × 时间
- 计算：60 × 2.5 = 150
- 单位：公里
  答案：150公里

【待解】
问题：工程队3天修完公路的1/4，照此速度，修完全程需几天？
推理：
```

### 7.3 推理链设计要点

| 要素              | 作用         | 示例                                  |
| ----------------- | ------------ | ------------------------------------- |
| **显式步骤标记**  | 结构化分解   | "第一步：""第二步：" 或 "- " 项目符号 |
| **中间计算展示**  | 防止跳步错误 | "5 - 2 = 3" 而非直接 "剩下3个"        |
| **公式/原理引用** | 激活领域知识 | "根据勾股定理...""由需求定律可知..."  |
| **单位/边界检查** | 培养严谨性   | "检查：3天是1/4，则全程需 3×4=12天"   |
| **结论明确分离**  | 方便提取答案 | 独立一行 "答案：XX"                   |

## 8. 四种技术对比

| 技术              | 示例内容       | 模型学到什么       | 适用复杂度  | 成本             |
| ----------------- | -------------- | ------------------ | ----------- | ---------------- |
| **Zero-Shot**     | 无             | 依赖预训练知识     | ⭐ 简单     | 最低             |
| **Zero-Shot CoT** | 无，加触发句   | 激活内置推理能力   | ⭐⭐ 中等   | 低               |
| **Few-Shot**      | 输入→答案对    | 任务格式和映射模式 | ⭐⭐ 中等   | 中（示例token）  |
| **Few-Shot CoT**  | 输入→推理→答案 | 推理步骤和解题策略 | ⭐⭐⭐ 复杂 | 高（推理链很长） |

## 9. 效果对比：同一问题

**问题：**

> 水库有进水闸和排水闸。单开进水闸5小时灌满，单开排水闸3小时排完。同时开两闸，几小时灌满？

| 方法                   | 模型输出                                                                    | 准确率  |
| ---------------------- | --------------------------------------------------------------------------- | ------- |
| **Zero-Shot**          | "8小时"（错误，简单相加）                                                   | ❌      |
| **Zero-Shot CoT**      | 尝试列方程但混淆进排水关系                                                  | ⚠️ 50%  |
| **Few-Shot**（无推理） | "7.5小时"（错误，平均）                                                     | ❌      |
| **Few-Shot CoT**       | 正确建模：设总量为1，净效率 1/5-1/3=-2/15，指出实际排干而非灌满，或修正理解 | ✅ 90%+ |

## 10. 高级技巧

### 技巧1：多样化推理路径

```markdown
【示例A】代数方程法
【示例B】赋值具体数字法
【示例C】画图/列表法
```

→ 模型学会一题多解，选择最优路径

### 技巧2：错误示例+纠正（对比学习）

```markdown
【错误示例】
问题：...
错误推理：...（此处故意漏看条件）
错误答案：X

【正确示例】
问题：相同问题
正确推理：注意检查...（指出上例遗漏）
正确答案：Y
```

→ 模型学会避坑，自我验证

### 技巧3：动态示例选择（RAG增强）

根据用户问题，从示例库检索最相似的3个CoT示例插入Prompt

→ 示例始终相关，避免无关模式干扰

## 11. 场景选择决策树

```
任务复杂度评估
    │
    ├─ 模型预训练已覆盖（常识/简单任务）
    │   └─ Zero-Shot 足够
    │
    ├─ 需要推理但模式标准（数学/逻辑）
    │   ├─ 单步推理 → Zero-Shot CoT
    │   └─ 多步复杂 → Few-Shot CoT（3-5例）
    │
    ├─ 特定格式/风格任务（翻译特定术语、仿写）
    │   └─ Few-Shot（无需CoT，教模式即可）
    │
    └─ 专业领域+复杂推理（医疗诊断、法律分析）
        └─ Few-Shot CoT + 角色设定 + 知识检索
```

## 12. 关键洞察

| 要点                    | 说明                                     |
| ----------------------- | ---------------------------------------- |
| **CoT 示例质量 > 数量** | 1个清晰完整的推理链 > 3个潦草答案        |
| **示例难度匹配**        | 示例过难，模型畏难；过易，模型轻视真问题 |
| **自一致性验证**        | 同一问题换示例顺序问多次，投票最稳答案   |
| **示例可学习**          | 用模型生成的高质CoT作为新示例，自动扩充  |
| **长示例吃上下文**      | 复杂CoT示例占token多，总示例数需减少     |

## 13. 四种技术总结对比

| 技术             | 英文                | 核心机制           | 适用场景                 | 成本 |
| ---------------- | ------------------- | ------------------ | ------------------------ | ---- |
| **零样本提示**   | Zero-Shot Prompting | 直接提问，不给示例 | 简单任务，模型已具备能力 | 最低 |
| **零样本思维链** | Zero-Shot CoT       | 加触发句激活推理   | 中等复杂度推理           | 低   |
| **少样本提示**   | Few-Shot Prompting  | 示例教模式         | 特定格式/风格任务        | 中   |
| **少样本思维链** | Few-Shot CoT        | 示例教推理策略     | 复杂推理任务             | 高   |

---

> 🔥 **一句话总结**
>
> **Zero-Shot CoT**：告诉模型"请思考"
> **Few-Shot CoT**：展示给模型"如何思考"
>
> 后者是手把手教解题策略，成本最高，效果最强，是攻克复杂推理任务的终极武器。
