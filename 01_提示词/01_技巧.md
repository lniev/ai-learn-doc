
### 角色扮演

- 你现在是xxx，你要xx，到达xxx  + 具体限制

### 提示词核心技巧

| 元素 | 描述 | 例子 | 备注 |
| ---- | ---- | -------- | -------- |
| 指令 | 想要执行的特定任务 | 生成代码，翻译，解释，解析，分析 | |
| 场景 | 外部信息补充的信息告诉大模型更改的执行 | 你是一个前端开发人员，熟练使用react | 角色扮演 |
| 输入 | 输入内容或问题或者上下文，模型需要接收处理的东西 | 需要推理的问题 | |
| 输出 | 指定的输出格式或者类型 | 参考xx代码一样，生成列表代码 1000个字 md格式 | |

### 10 个提升学习效率的 GPT 指令模板**

---

### 1. 概念拆解指令（费曼学习法）

```
请用费曼学习法帮我彻底理解 [插入概念]：
1. 用简单语言解释这个概念（像教给10岁小孩）
2. 找出我理解中的知识盲区并指出
3. 用代码/图表/类比三种方式重新表达
4. 列出这个概念最容易混淆的3个误区
```

**适用场景**：遇到难以理解的技术概念（如 LCEL、StateGraph、Embeddings）

---

### 2. 错题归因指令

```
我在这道题/这段代码上犯了错：[粘贴错误内容]
请帮我做错误归因分析：
- 根本原因是什么？（概念不清/粗心/知识盲区）
- 属于哪种错误类型？（语法/逻辑/架构/最佳实践）
- 给我3个同类练习题巩固
- 总结一个"防错检查清单"让我以后自查
```

**适用场景**：代码报错、面试题做错、架构设计失误

---

### 3. 知识图谱构建指令

```
我正在学习 [主题]，请将以下零散知识点连接成知识图谱：
[粘贴知识点列表或文档内容]

要求：
- 用 Mermaid 语法生成层级关系图
- 标出核心概念 vs 衍生概念
- 指出哪些是前置知识必须掌握，哪些是进阶内容
- 填补知识点之间的逻辑断层
```

**适用场景**：整理 LangChain 各模块关系、复习阶段构建体系

---

### 4. 主动回忆测试指令

```
基于我们刚才讨论的 [主题]，生成10道测试题：
- 4道概念理解题（检查基础）
- 3道代码分析题（给出代码片段问输出/作用）
- 2道场景应用题（如何选型/设计）
- 1道综合设计题（完整架构设计）

不要直接给答案，等我回答后再批改并解释。
```

**适用场景**：学完一个章节后的自我检测、考前复习

---

## 5. 代码解释与优化指令

```
请解释这段代码，要求：
[粘贴代码]

1. 逐行注释说明（中文）
2. 指出使用了哪些设计模式/最佳实践
3. 标记哪些部分可以优化（性能/可读性/类型安全）
4. 给出 TypeScript 类型定义优化版本
5. 说明如果要用在 LangGraph 中需要如何改造
```

**适用场景**：阅读源码、Review 他人代码、优化自己的项目

---

### 6. 间隔重复卡片指令

```
将以下内容转换成 Anki/Quizlet 记忆卡片格式：
[粘贴学习材料]

要求：
- 正面：问题或概念名称（简洁）
- 背面：答案 + 代码示例 + 易错提示
- 标记难度等级（简单/中等/困难）
- 按艾宾浩斯遗忘曲线建议复习时间点

输出为 CSV 格式，方便导入。
```

**适用场景**：背诵 API 参数、设计模式、面试八股文

---

### 7. 学习路径动态规划指令

```
我当前水平：[初级/中级/高级]
目标：[具体目标，如掌握 LangGraph 多 Agent 编排]
可用时间：[每天1小时，共3周]

请制定动态学习计划：
- 将目标拆解为每日任务（具体到要读哪几页/写哪段代码）
- 每个节点设置 checkpoint（验收标准）
- 预留20%缓冲时间应对难点
- 如果第X天没完成，给出压缩方案（哪些内容可以跳过/延后）
```

**适用场景**：备考、技术转型、项目冲刺学习

---

### 8. 跨知识迁移指令

```
我已经精通 [已有技能，如 Python/Pandas]，现在要学 [新技能，如 TypeScript/LangChain]。

请用对比学习法：
1. 列出两者概念映射表（旧知识→新知识）
2. 指出思维模式差异（容易踩的坑）
3. 给出"如果我会 X，那么学 Y 只需要关注 Z"的简化路径
4. 生成一个对照表：Python 实现 vs TypeScript 实现
```

**适用场景**：技术栈迁移（如 Python→TS、React→Vue、SQL→NoSQL）

---

### 9. 深度追问苏格拉底指令

```
请用苏格拉底式提问法帮我深入理解 [主题]：
不要直接给答案，而是通过连续提问引导我思考：
- 为什么需要这个技术？没有它会怎样？
- 这个方案的trade-off是什么？
- 边界情况在哪里？什么场景不适用？
- 如果资源限制（内存/延迟）如何妥协？
- 5年后这个技术可能如何演进？

每问一步等待我的回答，再引导下一步。
```

**适用场景**：技术选型、架构设计、准备技术演讲/写作

---

### 10. 多模态输出指令

```
请将 [复杂概念，如 RAG 流程/LangGraph 状态机] 转换成多种学习材料：

1. 文字版：步骤说明（适合阅读）
2. 图表版：用 Mermaid/ASCII 画图（适合视觉记忆）
3. 代码版：最小可运行示例（适合动手）
4. 比喻版：用生活中的比喻解释（适合记忆）
5. 教学版：假设我要给同事做分享，生成10分钟演讲提纲

确保四种方式信息一致，互相补充而非重复。
```

**适用场景**：攻克难点、准备技术分享、写博客文档

---

### 💡 使用建议

| 学习阶段 | 推荐指令组合 |
|---------|------------|
| **初学** | 1 + 3 + 7（理解概念→建立体系→规划路径）|
| **练习** | 2 + 4 + 5（错题分析→主动测试→代码优化）|
| **复习** | 6 + 8 + 9（记忆卡片→知识迁移→深度思考）|
| **输出** | 10（教学相长，用输出倒逼输入）|

**提示**：这些指令可以组合使用，比如在学 LangChain 时先用指令 7 制定计划，遇到难懂的 Agents 概念用指令 1，看完文档用指令 4 自测，写代码遇到 Bug 用指令 2 复盘。
