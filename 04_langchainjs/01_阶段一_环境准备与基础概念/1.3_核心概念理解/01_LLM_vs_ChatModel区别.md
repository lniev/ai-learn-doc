# SFT 和 RLHF：大模型训练中的关键技术

## 概览

| 缩写     | 全称                                       | 中文                   | 作用                       |
| -------- | ------------------------------------------ | ---------------------- | -------------------------- |
| **SFT**  | Supervised Fine-Tuning                     | 监督微调               | 教会模型"怎么说人话"       |
| **RLHF** | Reinforcement Learning from Human Feedback | 基于人类反馈的强化学习 | 让模型"说得更符合人类偏好" |

> 💡 **训练流程**：预训练 → **SFT** → **RLHF**

---

## 1. SFT (监督微调)

### 是什么？

用人工标注的高质量对话数据，以**监督学习**的方式微调基础 LLM。

### 数据格式示例

```json
[
	{
		"instruction": "解释什么是光合作用",
		"input": "",
		"output": "光合作用是绿色植物利用光能..."
	},
	{
		"messages": [
			{ "role": "user", "content": "你好" },
			{ "role": "assistant", "content": "你好！有什么我可以帮您的吗？" }
		]
	}
]
```

### 核心作用

| 方面         | 效果                                      |
| ------------ | ----------------------------------------- |
| **格式对齐** | 让模型学会对话格式（角色标签、轮流发言）  |
| **指令遵循** | 学会理解并执行用户指令                    |
| **基础能力** | 获得初步的 helpful、honest、harmless 能力 |

### 特点

| 优点 ✅          | 缺点 ⚠️                                     |
| ---------------- | ------------------------------------------- |
| 训练稳定、效率高 | 数据质量决定上限（Garbage in, garbage out） |
|                  | 只能模仿"标准答案"，无法超越标注者水平      |

---

## 2. RLHF (基于人类反馈的强化学习)

### 是什么？

用**人类偏好数据**训练奖励模型，再通过**强化学习**优化策略模型。

### 三步流程

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│  第1步：训练奖励模型  │  →  │  第2步：生成+打分    │  →  │  第3步：强化学习优化  │
│  (Reward Model)  │     │  (生成多个回答)      │     │  (PPO 算法)       │
└─────────────────┘     └─────────────────┘     └─────────────────┘
```

### 详细步骤

| 步骤 | 内容                  | 说明                                                              |
| ---- | --------------------- | ----------------------------------------------------------------- |
| ①    | **收集偏好数据**      | 同一问题，模型生成多个回答，人类标注"哪个更好" → A > B > C 的排序 |
| ②    | **训练奖励模型 (RM)** | 学习预测人类偏好分数 → 输入：(prompt, response) → 输出：score     |
| ③    | **PPO 优化**          | 用奖励模型打分，通过 PPO 算法更新策略模型 → 高分回答↑ 低分回答↓   |

### 核心优势

| 能力               | 说明                                 |
| ------------------ | ------------------------------------ |
| **超越标注者**     | 模型可以探索比人工示范更好的回答方式 |
| **对齐人类价值观** | 学会"什么是有用的、无害的、诚实的"   |
| **处理模糊场景**   | 在复杂权衡中做出更符合人类期望的选择 |

---

## SFT vs RLHF 对比

| 维度         | SFT                  | RLHF                               |
| ------------ | -------------------- | ---------------------------------- |
| **学习方式** | 模仿学习（复制示范） | 强化学习（探索优化）               |
| **数据需求** | 高质量完整对话       | 偏好对比数据（成对比较）           |
| **训练成本** | 较低                 | 高（需要奖励模型+PPO）             |
| **效果上限** | 受限于标注者水平     | 可能超越标注者                     |
| **稳定性**   | 稳定                 | 可能出现奖励黑客（Reward Hacking） |
| **主要解决** | 格式、基础能力       | 偏好对齐、细微质量提升             |

---

## 实际应用中的变体

| 技术                                     | 说明                             | 代表                       |
| ---------------------------------------- | -------------------------------- | -------------------------- |
| **DPO** (Direct Preference Optimization) | 直接用偏好数据优化，无需奖励模型 | 简化版 RLHF，更高效        |
| **RLAIF**                                | 用 AI 反馈代替人类反馈，降低成本 | Constitutional AI (Claude) |
| **KTO**                                  | 二元反馈（好/坏）优化            | 比 DPO 更数据高效          |

---

## 总结图示

```
基础模型 (GPT/LLaMA)
    ↓
[SFT] 监督微调：学会对话格式和指令遵循
    ↓
[RLHF] 强化学习对齐：优化人类偏好，提升 helpful/harmless/honest
    ↓
Chat Model (ChatGPT/Claude/Kimi)
```

> 📌 **一句话记忆**：SFT 教模型"怎么说话"，RLHF 教模型"说什么话让人类更满意"。

# LLM 和 Chat Model 的区别

## 核心区别

| 维度           | LLM (大语言模型)                               | Chat Model (对话模型)                                   |
| -------------- | ---------------------------------------------- | ------------------------------------------------------- |
| **定义**       | 通用的大型语言模型，专注于文本生成、理解和推理 | 专门优化用于对话交互的模型                              |
| **训练目标**   | 预测下一个 token，学习语言模式和知识           | 在 LLM 基础上，使用对话数据微调，学习对话格式和交互能力 |
| **输入输出**   | 接受任意文本，输出连续文本                     | 接受结构化对话（如消息列表），输出符合对话语境的回复    |
| **上下文处理** | 通常处理单一文本序列                           | 管理多轮对话历史，理解角色（用户/助手）                 |

---

## 具体对比

### 1. 架构层面

- **LLM**：基础架构（如 GPT、LLaMA），核心是 Transformer 解码器
- **Chat Model**：通常是 **LLM + 对话微调（SFT/RLHF）**，例如 GPT-3.5 → ChatGPT

### 2. 使用方式

```
【LLM 典型用法】
输入："翻译这句话：Hello world"
输出："你好，世界"

【Chat Model 典型用法】
输入：[
  {"role": "system", "content": "你是翻译助手"},
  {"role": "user", "content": "Hello world"}
]
输出："你好，世界"
```

### 3. 能力特点

| 能力        | LLM             | Chat Model           |
| ----------- | --------------- | -------------------- |
| 开放式生成  | ✅ 强           | ✅ 强                |
| 遵循指令    | ⚠️ 一般         | ✅ 强                |
| 多轮对话    | ❌ 弱（无记忆） | ✅ 强                |
| 安全性/对齐 | ⚠️ 较低         | ✅ 较高（经过 RLHF） |
| 角色一致性  | ❌ 弱           | ✅ 强                |

```
基础 LLM (预训练)
    ↓
指令微调 (Instruction Tuning)
    ↓
对话微调 (对话数据 + RLHF)
    ↓
Chat Model (如 GPT-4、Claude、Kimi)
```

## 实际例子

| 模型             | 类型       | 说明                         |
| ---------------- | ---------- | ---------------------------- |
| GPT-3            | LLM        | 基础大模型，文本补全为主     |
| GPT-3.5-turbo    | Chat Model | 针对对话优化                 |
| LLaMA-2-70B      | LLM        | Meta 开源基础模型            |

| Kimi 当前版本    | Chat Model | 专为多轮对话和超长上下文优化 |

---

## 总结

> 💡 **一句话理解**：Chat Model 是 LLM 的"对话特化版"。就像一个是通用人才，一个是客服专家——后者在前者基础上，专门训练了如何与人交流、理解意图、保持上下文连贯等能力。

在实际应用中：

- 构建**聊天机器人、助手类产品** → 使用 **Chat Model**
- **文本分析、批量生成**等任务 → 基础 **LLM** 可能更灵活高效
