02 消息类型（Message Types）

核心概念

Chat Model 使用结构化消息而非纯文本字符串。每种消息类型代表对话中的不同角色。

```
对话结构：
┌─────────────────┐
│  SystemMessage  │  ← 设定 AI 身份和行为规则（可选）
├─────────────────┤
│  HumanMessage   │  ← 用户输入
├─────────────────┤
│  AIMessage      │  ← AI 回复
├─────────────────┤
│  HumanMessage   │  ← 用户追问
├─────────────────┤
│  AIMessage      │  ← AI 再次回复
└─────────────────┘
```

---

基础消息类型

```javascript
import { ChatOpenAI } from "@langchain/openai";
import {
  SystemMessage,
  HumanMessage,
  AIMessage
} from "@langchain/core/messages";

const model = new ChatOpenAI({ modelName: "gpt-4o-mini" });

// 1. SystemMessage：设定 AI 角色和规则（最高优先级）
const systemMsg = new SystemMessage(
  "你是一位专业的 JavaScript 导师，用简洁的中文回答，避免使用专业术语。"
);

// 2. HumanMessage：用户输入
const humanMsg = new HumanMessage("什么是闭包？");

// 3. AIMessage：AI 的回复（通常用于多轮对话历史）
const aiMsg = new AIMessage("闭包是函数记住并访问其外部作用域变量的能力。");

// 组合调用（多轮对话）
const response = await model.invoke([
  systemMsg,
  humanMsg
]);

console.log(response.content);
```

---

完整消息类型清单

消息类型	用途	使用场景
`SystemMessage`	设定 AI 身份、规则、限制	定义助手角色、输出格式要求
`HumanMessage`	用户输入	用户提问、指令
`AIMessage`	AI 回复	历史记录、少样本示例
`FunctionMessage`	函数执行结果（Legacy）	旧版工具调用结果
`ToolMessage`	工具执行结果（新版）	Agent 中工具返回的数据

实际应用示例

示例 1：带角色的对话系统

```javascript
import { ChatOpenAI } from "@langchain/openai";
import { SystemMessage, HumanMessage } from "@langchain/core/messages";

const model = new ChatOpenAI({ temperature: 0.7 });

const messages = [
  new SystemMessage(`
    你是"代码审查助手"，职责是：
    1. 检查代码中的潜在 Bug
    2. 提出性能优化建议
    3. 用 🐛 标记 Bug，用 ⚡ 标记优化点
    4. 保持礼貌和建设性
  `),
  new HumanMessage(`
    请审查以下代码：
    \`\`\`javascript
    function getData() {
      fetch('/api/data')
        .then(res => res.json())
        .then(data => console.log(data))
    }
    \`\`\`
  `)
];

const result = await model.invoke(messages);
console.log(result.content);
// 输出带 🐛 和 ⚡ 标记的审查意见
```

示例 2：少样本学习（Few-Shot）

```javascript
import { ChatOpenAI } from "@langchain/openai";
import {
  SystemMessage,
  HumanMessage,
  AIMessage
} from "@langchain/core/messages";

const model = new ChatOpenAI({ temperature: 0 });

// 通过历史消息给 AI 示例
const messages = [
  new SystemMessage("你是一个情感分析器，输出格式：情感：正面/负面/中性 | 置信度：0-1"),

  // 示例 1
  new HumanMessage("这部电影太棒了！"),
  new AIMessage("情感：正面 | 置信度：0.95"),

  // 示例 2
  new HumanMessage("服务态度很差"),
  new AIMessage("情感：负面 | 置信度：0.88"),

  // 实际输入
  new HumanMessage("价格还可以，但是配送太慢了")
];

const result = await model.invoke(messages);
console.log(result.content);
// 输出：情感：负面 | 置信度：0.75（或类似）
```

示例 3：多轮对话（带历史记录）

```javascript
import { ChatOpenAI, HumanMessage, AIMessage } from "@langchain/openai";

const model = new ChatOpenAI();

// 模拟已存在的对话历史
const conversationHistory = [
  new HumanMessage("你好，我叫张三"),
  new AIMessage("你好张三！很高兴认识你。有什么我可以帮助你的吗？"),
  new HumanMessage("记住我的名字"),
  new AIMessage("好的，我记住了你叫张三。")
];

// 继续对话（模型能从历史中获取上下文）
const newMessage = new HumanMessage("我叫什么名字？");
const response = await model.invoke([...conversationHistory, newMessage]);

console.log(response.content); // "你叫张三。"
```

消息内容的其他格式

多模态内容（文本 + 图片）

```javascript
import { HumanMessage } from "@langchain/core/messages";

const imageMessage = new HumanMessage({
  content: [
    { type: "text", text: "描述这张图片" },
    {
      type: "image_url",
      image_url: { url: "https://example.com/image.jpg" }
    }
  ]
});

// 或使用 base64
const base64Image = new HumanMessage({
  content: [
    { type: "text", text: "分析这张图表" },
    {
      type: "image_url",
      image_url: { url: "data:image/png;base64,iVBORw0KGgo..." }
    }
  ]
});
```

与 LCEL 链式语法结合

```javascript
import { ChatPromptTemplate } from "@langchain/core/prompts";
import { ChatOpenAI } from "@langchain/openai";

const model = new ChatOpenAI();

// ChatPromptTemplate 内部自动处理消息类型
const prompt = ChatPromptTemplate.fromMessages([
  ["system", "你是{role}，用{style}风格回答"],
  ["human", "{question}"]
]);

const chain = prompt.pipe(model);

const result = await chain.invoke({
  role: "技术专家",
  style: "幽默",
  question: "什么是递归？"
});
// 等效于手动构建 SystemMessage + HumanMessage
```
