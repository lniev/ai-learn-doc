## 14. 自洽性提示（Self-Consistency Prompting）

### 14.1 核心概念

自洽性提示 = 对同一问题生成**多条独立推理路径**，通过**投票/聚合**选出最可靠答案

| 方法     | 流程                               | 特点       |
| -------- | ---------------------------------- | ---------- |
| 传统 CoT | 生成1条推理链 → 1个答案            | 可能出错   |
| 自洽性   | 生成N条推理链 → N个答案 → 多数表决 | 更可能正确 |

### 14.2 工作流程

```
输入问题
│
├─ 采样1（Temp=0.8）→ 推理路径A → 答案：42
├─ 采样2（Temp=0.9）→ 推理路径B → 答案：42
├─ 采样3（Temp=0.7）→ 推理路径C → 答案：38
├─ 采样4（Temp=0.85）→ 推理路径D → 答案：42
└─ 采样5（Temp=0.75）→ 推理路径E → 答案：42
│
▼
答案统计：42出现4次，38出现1次
最终输出：42（置信度80%）
```

### 14.4 应用场景

- 数学推理问题
- 逻辑分析任务
- 复杂决策制定
- 需要高可靠性的问答场景

---

> 🔥 **一句话总结**
>
> 自洽性提示 = 多条推理链 + 多数表决
>
> 用"群体智慧"提升模型输出的可靠性，是CoT技术的进阶应用。
